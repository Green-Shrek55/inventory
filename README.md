## Информационная система учета оборудования

Spring Boot (3.3) приложение для ведения инвентаризации компьютеров и периферии внутри организации. Реализована полная админ‑панель и отдельные рабочие пространства для IT‑службы и экономиста: управление пользователями и ролями, справочники оргструктуры, учет/архивирование техники, заявки на ремонт, финансовая аналитика, журнал действий, резервное копирование и готовый SQL‑скрипт для переноса в PostgreSQL.

### Как запустить

```bash
mvn spring-boot:run
```

- По умолчанию используется H2 (in-memory) в режиме совместимости PostgreSQL.
- Для работы с реальной БД PostgreSQL создайте базу `inventory_db` и запустите приложение с профилем `postgres`:
  ```bash
  mvn spring-boot:run -Dspring-boot.run.profiles=postgres
  ```

### Учетные записи

| Логин      | Пароль     | Роль       |
|------------|------------|------------|
| `admin`    | `admin`    | ADMIN      |
| `it`       | `itpass`   | IT         |
| `economist`| `economist`| ECONOMIST  |

Пароли можно изменить в админке (`/admin/users`).

### Админка

- управление пользователями и ролями (создание, смена роли/подразделения, сброс пароля, блокировка, удаление);
- управление оргструктурой (подразделения, сотрудники, локации, типы оборудования);
- просмотр журнала действий и управление резервными копиями. Все операции с устройствами вынесены в IT-панель.

### IT панель (`/it`)

- полный CRUD по устройствам (отдельный раздел `/it/equipment`) + перераспределение по отделам/сотрудникам и перемещение на склад;
- управление складом, включая учёт находящихся там устройств и выдачу обратно отделам;
- инвентаризация склада `/it/inventory`: сканирование по коду (инвентарному номеру) с фиксацией времени сканирования;
- управление заявками на ремонт (создание, смена статуса, назначение исполнителей, фиксация комментариев);
- быстрые метрики по количеству оборудования и сервисных обращений.

### Панель экономиста (`/economist`)

- агрегированные показатели стоимости активов по отделам и локациям;
- список последних закупок и топ дорогих единиц с учетом стоимости;
- быстрый доступ к аналитике для оценки бюджета и остаточной стоимости.

### SQL‑скрипт для PostgreSQL

`src/main/resources/db/inventory-schema.sql` — самодостаточный DDL/DML файл:

1. Создает все таблицы и ограничения, совместимые с H2/PostgreSQL (включая стоимость оборудования и заявки на обслуживание).
2. Наполняет справочники, сотрудников, учетные записи (`admin`, `it`, `economist`) с bcrypt‑паролями.
3. Добавляет примеры оборудования, заявок на ремонт и действий.

Использование:

```bash
psql -U postgres -d inventory_db -f src/main/resources/db/inventory-schema.sql
```

После выполнения можно запускать приложение с профилем `postgres` — схема и данные уже будут на месте.
